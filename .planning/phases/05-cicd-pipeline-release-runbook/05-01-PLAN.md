<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| Tag push (developer/agent) -> GitHub Actions runner | A pushed tag matching `v*.*.*` causes GitHub to schedule and run this repo-defined workflow on a `windows-latest` runner — the runner executes exactly what this YAML file specifies, nothing more. |
| GitHub Actions runner -> GitHub Releases (`contents: write`) | The runner's last step publishes a public-facing release artifact using the auto-provided, explicitly-scoped `GITHUB_TOKEN`. |
| Third-party Action (`softprops/action-gh-release@v2`) -> repo's `GITHUB_TOKEN` | Supply-chain surface: a compromised or re-targeted mutable version tag on this community Action could, in principle, run arbitrary code with `contents: write`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Elevation of Privilege | `.github/workflows/release.yml` `permissions` block | mitigate | Explicit `permissions: contents: write` only (never `write-all`) — repo default is already `read` (verified via `gh api repos/tpougy/finance-fmt-tools/actions/permissions/workflow`), so this workflow-level override grants exactly what release publishing needs and nothing more. |
| T-05-02 | Tampering | `softprops/action-gh-release@v2` mutable version tag | accept | Pinned to the major-version tag `@v2`, matching GitHub's own documented convention and the sibling project's proven, unmodified, currently-working pattern (verified via a real successful run 6 days before this research). SHA-pinning is optional, stricter hardening — not required by CLAUDE.md/REQUIREMENTS.md and not used by the sibling either; documented residual risk, not blocking. |
| T-05-03 | Tampering | Release zip filename could accidentally be versioned, breaking `scripts/install.ps1`'s trust in a stable download URL | mitigate | Task 1's acceptance criteria explicitly gate on the literal, non-interpolated `FinanceFmtTools.zip` filename (05-RESEARCH.md Pitfall 2). |
| T-05-SC | Tampering | GitHub Actions Marketplace references (`actions/checkout`, `actions/setup-dotnet`, `softprops/action-gh-release`) | accept | Not npm/pip/cargo packages, so the standard slopcheck/package-legitimacy gate does not apply directly; legitimacy already manually audited in 05-RESEARCH.md's Package Legitimacy Audit (all 3 approved, cross-referenced against a real, recent, successful production run in the sibling repo under the same GitHub account) — no additional blocking checkpoint required here. |

</threat_model>
