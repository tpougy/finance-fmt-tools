<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| uninstall.ps1 -> HKCU registry | Local, per-user privilege boundary — removal only, no elevation |
| uninstall.ps1 -> shared Resiliency key | Multiple add-ins may share `DoNotDisableAddinList`; removal must be scoped to this add-in's own named value only |
| verify-environment.ps1 -> local machine state | Read-only — no trust boundary crossed, no writes of any kind |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering | Resiliency key removal (`DoNotDisableAddinList`) | mitigate | `Remove-ItemProperty -Name $ProgId` only, never `Remove-Item` on the parent key — other add-ins' entries under the same key are preserved |
| T-04-06 | Denial of Service (self-inflicted) | File removal loop in uninstall.ps1 | mitigate | Iterates the exact named `$AllFiles` list and deletes only those; never a recursive delete of `$InstallDir` unless it is already empty afterward |
| T-04-07 | Information Disclosure | verify-environment.ps1 output | accept | Output is local-machine diagnostic text (OS version, Office bitness, tool presence) intended for the same user running the script; no secrets are read or displayed |
| T-04-SC | Tampering | npm/pip/cargo/NuGet installs | accept | Not applicable — this phase introduces zero new packages (04-RESEARCH.md Package Legitimacy Audit) |
</threat_model>
