## Tech Debt

See YAML frontmatter `tech_debt` block above for the structured list. In prose:

1. **Phase 3 — live-Excel smoke test (`human_needed`)**: This Linux/WSL sandbox has no Windows/Excel/COM runtime. The itemized 8-point checklist (Ribbon tab appearance, all 12 format buttons, invalid-selection message, both checkboxes' session-only behavior, About/docs links) is recorded verbatim in `03-02-SUMMARY.md` and `03-VERIFICATION.md` — not faked, not assumed passing.
2. **Phase 4 — live install/uninstall test (`human_needed`)**: Same environment constraint. The itemized 8-step checklist (fresh install, Ribbon tab appears, idempotent re-install, the Resiliency behavioral test, uninstall, idempotent uninstall) is recorded verbatim in `04-03-SUMMARY.md` and `04-VERIFICATION.md`.
3. **Phase 5 — real release push (`human_needed`, authorization boundary)**: Unlike the two items above, `git`/`gh` access does exist in this sandbox — the reason this is deferred is that pushing ~95 local migration commits to the real, public `tpougy/finance-fmt-tools` repo for the first time, and cutting a real GitHub Release, are irreversible-ish, externally-visible actions that this project's own operating rules require explicit human authorization for, regardless of how autonomous the rest of the milestone's execution has been. The itemized 9-step checklist (review changelog, choose tag, bump version fields, push `main`, push `archive/vba-legacy`, tag+push, watch CI or run the manual `gh release create` fallback, confirm, optional live-Excel re-test) is recorded verbatim in `05-04-SUMMARY.md` and `05-VERIFICATION.md`. Resume-signal on file: `"deferred — not releasing yet"`.
4. **Minor UX polish (non-blocking)**: `AddInHost.ApplyFormat`'s `_gateway == null` guard (a defensive check for a Ribbon callback somehow firing before `OnConnection`/`Wire` completes — practically unreachable in Excel's real add-in lifecycle, since Excel always calls `OnConnection` before any Ribbon button exists to click) logs a warning but shows no `MessageBox`, unlike the sibling FMT-06 invalid-selection guard a few lines below it. Flagged by the integration checker; accepted as tech debt rather than fixed speculatively, since there is no live Excel session available in this environment to validate a change to this exact code path, and the scenario it guards against is not reachable through Excel's documented add-in lifecycle.
