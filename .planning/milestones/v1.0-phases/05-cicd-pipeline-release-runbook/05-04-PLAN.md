<objective>
Perform a final automated integration check across all of Phase 5's local deliverables (05-01, 05-02,
05-03), then hand off — via a hard blocking checkpoint — the exact, complete, ordered command sequence
required to actually push `main`, push `archive/vba-legacy`, tag a release, and either let the real
`windows-latest` GitHub Actions workflow fire or run the manual `gh release create` runbook against the
real `tpougy/finance-fmt-tools` remote.

Purpose: `git push origin main`, `git push origin archive/vba-legacy`, a real tag push, and
`gh release create` against the real remote are irreversible-ish, externally-visible actions on shared
state (this repo is public, already has 2 real legacy VBA releases visible to users, and local `main`
is 79 commits ahead of `origin/main` — none of this milestone's C# migration has ever been pushed).
Per this project's explicit operating constraint for this phase, these specific actions MUST NOT be
autonomously executed by an execution agent, regardless of how autonomous the rest of this milestone's
workflow has been. This plan's only job is to verify everything is ready, then request explicit human
authorization for the one step this phase cannot (and should not) do on its own.
Output: A verified-ready local state, and a recorded human decision/outcome in this plan's SUMMARY.md.
</objective>
