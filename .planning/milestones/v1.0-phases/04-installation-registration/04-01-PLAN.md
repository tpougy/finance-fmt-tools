## Trust Boundaries

| User-supplied `-Package`/`-Source` path -> extraction/copy logic | Untrusted local path input; must be validated before use, and extraction must never target the final install directory directly |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | GitHub Releases zip download (`Invoke-WebRequest`) | mitigate | TLS 1.2 forced via `ServicePointManager` before any network call; HTTPS-only `github.com/.../releases/latest/download/` URL, no plaintext fallback |
| T-04-02 | Tampering | Zip extraction (`Expand-Archive`) | mitigate | Extract only into a freshly created, uniquely-named `%TEMP%` subfolder, never directly into `$InstallDir`; only the 4 named files are copied out by exact filename, never a blanket directory copy |
| T-04-03 | Spoofing | The documented `irm \| iex` one-liner's source URL | accept | The script cannot control which URL a user pastes; the canonical `raw.githubusercontent.com/tpougy/finance-fmt-tools/main/scripts/install.ps1` URL is the only one documented in this script's own comment-based help |
