<objective>
Remove the `PadHyphen` ("_-") token from the accounting number-format string produced by
`AccountingFormatBuilder.Build`, so each of the 3 sections (positive/negative/zero) ends at
`CloseParenPad` ("_)") or the literal closing paren (")") instead of being followed by a trailing
hyphen-width pad. This is a real behavior change to the generated Excel number-format string, not a
pure refactor.

Purpose: The trailing "_-" pad was inherited from the VBA original but is no longer wanted in the
final format string — dropping it changes what Excel actually renders/aligns for every Fin format
(FMT_FIN_2D/4D/8D and the integer variant), across all decimals x forceAlign x zeroDash combinations.

Output: `AccountingFormatBuilder.cs` with all 12 `+ FormatTokens.PadHyphen` concatenations removed,
`FormatTokens.cs` with the now-dead `PadHyphen` constant deleted, and
`AccountingFormatBuilderTests.cs` updated to assert the new hyphen-free expected strings for all 16
Theory rows.
</objective>
